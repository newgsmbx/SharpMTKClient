# Buyer Guide

SharpMTKClient is a private C# source code package for developers and companies that want to build their own MediaTek / MTK Windows software.

## Who It Is For

SharpMTKClient is designed for:

* C# / .NET developers.
* Software owners building MTK service tools.
* Research teams studying MTK protocol behavior.
* Companies that need internal device service workflows.
* Developers who need private branding, customization and licensing control.

## Who It Is Not For

SharpMTKClient is not intended for users who only want a simple ready-made executable.

It is also not intended for unauthorized device access, illegal servicing, data abuse or public redistribution of private protocol implementation.

## What You Receive

The licensed package may include, depending on the agreement:

* C# / .NET source code.
* Windows desktop project files.
* Protocol modules for supported MTK workflows.
* Flash, partition and file-system related source areas.
* NV, IMEI, RPMB and key-related development paths where included.
* Documentation and compile guidance.
* Minor updates and bug fixes during the support period.

## Before Purchase

Before purchasing, clarify:

* Required protocol family: LEGACY, XFLASH, XML or a combination.
* Target chipsets and brands.
* Required operations: flash, read, erase, file management, NV, RPMB, IMEI or patch workflows.
* Whether private customization is required.
* Whether compiled redistribution is required.
* Support period and update expectations.

## Device Support

MTK support is always dependent on chipset, boot mode, selected DA, security state, firmware package and target operation.

No source package can guarantee every operation on every device. Confirm critical target models before purchase if your workflow depends on specific brands or chipsets.

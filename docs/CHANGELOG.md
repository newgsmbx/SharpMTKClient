# Changelog

This changelog describes public documentation and product-positioning updates for this showcase repository.

## 2026-04-25

* Replaced the old public skeleton project with documentation-only product presentation.
* Removed public C# project files, resources, build artifacts and IDE files from the repository.
* Added commercial source-code positioning for SharpMTKClient.
* Added license terms, notice, security policy and buyer-facing documentation.
* Expanded public feature coverage for MTK protocol, flash, partition, file-system, NV, RPMB and research-oriented workflow areas.

## Product Feature History Highlights

The private SharpMTKClient package has included development paths and updates around:

* Patch cert workflows.
* IMEI information read from device paths such as `nvdata` and `ld0b`.
* IMEI repair workflow areas for selected Vivo, Xiaomi, OPPO, Infinix, Tecno, Itel and Samsung MTK paths.
* NV item decryption.
* Xiaomi Mi Account disable, remove and global conversion paths.
* Payjoy remove and Payjoy unlock paths.
* Huawei ID reset and OPPO ID reset paths.
* OTA remove paths for OPPO, Realme, OnePlus and Xiaomi.
* MDM remove, MDM + OTA remove, Walton Walock remove and Nothing / CMF SIM Lock / MDM remove paths.
* Xiaomi anti-relock modem patch paths.
* Persist patch, demo remove, restore Vbmeta and Vbmeta Patch V1 / V2 / V3 paths.
* Fix no efuse state, fix DM corrupted and remove security plugin paths.
* EROFS partition support paths.
* F2FS partition support paths.
* Raw and sparse image handling.
* MTK V6 / XML workflow updates.
* Carbonara workflow integration for selected supported devices.
* SLA / DAA handling updates.
* Android `build.prop` reading from supported partitions.
* EXT4 partition browsing, extraction and write-related workflows.
* Logical partition and `super` partition handling.
* RPMB read, write and erase paths.
* Vendor and model selection workflows.
* Selected Samsung, Motorola, Infinix, Honor, Vivo, Redmi and TCL support examples where included by package scope.

Exact feature availability depends on licensed package version, target device and agreement scope.

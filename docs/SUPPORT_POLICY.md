# Support Policy

This document describes the default support policy for licensed SharpMTKClient source code buyers.

## Default Support Period

Unless a separate written agreement says otherwise, licensed buyers receive one year of minor updates and bug-fix support from the purchase date.

Support is tied to the purchased package scope.

## Included Support

Default support may include:

* Build and compile guidance.
* Bug-fix updates for included modules.
* Minor compatibility updates.
* Clarification of documented workflows.
* Help diagnosing reproducible protocol or device workflow issues.

## Not Included by Default

The default support plan does not automatically include:

* Custom feature development.
* New brand-specific modules.
* New chipset research.
* Private loader or payload acquisition.
* Server-side licensing systems.
* UI redesign or branding work.
* Integration into third-party commercial systems.
* Support for unauthorized or unlawful use.

These can be discussed separately if required.

## Reporting Issues

When reporting an issue, include:

* Package version or delivery date.
* Visual Studio and Windows version.
* Target device model and chipset.
* Boot mode and selected operation.
* Selected DA or firmware package where relevant.
* Full logs and screenshots.
* Steps to reproduce.

Incomplete reports may delay support.

## Update Scope

Minor updates and bug fixes are included during the active support period.

Major new protocol families, large architecture changes, new commercial modules or private customer-specific workflows may require a separate agreement.

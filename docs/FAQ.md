# FAQ

## Is the full source code in this public repository?

No. This public repository is a product showcase and documentation repository only. The private source code package is delivered through direct licensing.

## Why was the old public skeleton removed?

The public skeleton did not represent the full commercial package. This repository now focuses on accurate product positioning, documentation, licensing expectations and buyer guidance.

## Is SharpMTKClient a compiled repair tool?

No. SharpMTKClient is positioned as a source-code product for developers and companies that want to build, customize and control their own MTK service software.

## What language is the project written in?

SharpMTKClient is a C# / .NET Windows desktop source-code project.

## Does it support MTK V6 / XML workflows?

The project includes MTK V6 / XML workflow paths where supported by the licensed package, target device, firmware package, boot mode and selected operation.

## Does it support XFLASH and legacy DA flows?

Yes, supported package areas can include LEGACY, XFLASH and XML workflow families. Exact coverage depends on device, DA compatibility, security state and purchased scope.

## Does it depend on Python or UsbDk?

SharpMTKClient is designed as a native C# / Windows project. Selected package paths may use Windows API communication and optional USB handling components depending on the workflow scope.

## Can every listed feature work on every device?

No. MTK behavior depends on chipset, boot mode, security state, selected DA, firmware package, storage type and target operation. Model-specific requirements should be confirmed before purchase.

## Can buyers resell the source code?

No. The private source package may not be resold, leaked, exchanged, uploaded publicly, gifted or shared with third parties.

## Can a buyer build a commercial executable from it?

Compiled product distribution depends on the written license agreement. Clarify branding, redistribution and licensing requirements before purchase.

## Is custom development available?

Custom development, private branding, activation systems, installers, model-specific workflows and server integration can be discussed separately.

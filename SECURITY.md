# Security Policy

## Intended Use

SharpMTKClient is intended for lawful development, research, maintenance and authorized service scenarios involving MediaTek / MTK devices.

Device workflows may interact with firmware, partitions, security states, identifiers, NV data, RPMB paths and other sensitive areas. Users and licensees are responsible for operating only on devices and data they are legally authorized to handle.

## Public Repository Scope

This public repository is a showcase and documentation repository only.

Please do not request protected source code, private loaders, payloads, authorization material, customer files or sensitive implementation details through public issues.

## Reporting Security Issues

If you believe you found a security issue related to SharpMTKClient documentation, product packaging, private delivery, licensing access or supported workflows, contact privately:

<https://t.me/GsmCoder>

When reporting, include:

* A clear description of the issue.
* Affected workflow or module area.
* Steps to reproduce where appropriate.
* Expected impact.
* Any relevant screenshots, logs or version details.

## Responsible Handling

Do not publish exploit details, private code excerpts, payloads, keys, bypass material or customer data in public channels.

Reports that include sensitive information should be shared privately only.
